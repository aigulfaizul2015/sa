————————————————————————————————
Задача
Сформировать user stories (пользовательские требования) на основе бизнес-требований.
User stories должны:
- отражать цели и потребности пользователей
- раскрывать способы достижения бизнес-результатов
- служить основой для последующей детализации в функциональные требования
————————————————————————————————
Инструкции по выполнению
1. Проанализируй бизнес-требования.
2. Определи ключевые роли пользователей.
3. Для каждого бизнес-требования сгенерируй несколько user stories.
4. Сформируй 10–20 user stories.
5. Проверь:
- каждая story отражает пользовательскую ценность
- нет технической реализации
- нет дублирования
————————————————————————————————
Формат результата
Представь результат в виде списка.
Формат: **US-<номер>. Как** <роль>, **я хочу** <действие>, **чтобы** <ценность>.
Детализация (обязательно)
**Ценность:** <какую пользу получает пользователь или бизнес>
**Связь:** <FR-xxx, BR-xxx или не задано>
**Приоритет (MoSCoW):** Must / Should / Could / Won’t
————————————————————————————————
Требования к формулировке
- Используй шаблон: Как <роль>, я хочу <что>, чтобы <зачем>
- Фокус: на цели пользователя, на ценности
Правила приоритизации (обязательно)
- Must — критично для достижения бизнес-цели
- Should — важно, но есть обходные пути
- Could — улучшение, не критично
- Won’t — осознанно не входит в текущий scope
————————————————————————————————
Ограничения
- Не писать функциональные требования
- Не писать критерии приемки
- Не описывать UI / API / архитектуру
- Не объединять несколько целей в одну story
————————————————————————————————
Критерии качества
- Story отражает реальную пользовательскую цель
- Есть ценность
- Есть связь с бизнес-требованием
- Story атомарна
- Story понятна без контекста
————————————————————————————————
Пример
**US-001. Как** менеджер отеля, **я хочу** видеть ошибки в данных бронирования до сохранения, **чтобы** избежать некорректных записей.
**Ценность:** снижение количества ошибок
**Связь:** BR-001
**Приоритет (MoSCoW):** Must
**US-002. Как** менеджер отеля, **я хочу** получать уведомление о дублирующихся бронированиях, **чтобы** избежать повторных записей.
**Ценность:** снижение количества дублей
**Связь:** BR-001
**Приоритет (MoSCoW):** Should
————————————————————————————————
Ограничение ответа
- Только список user stories
- Без пояснений вне структуры
- Любой текст вне формата — ошибка
