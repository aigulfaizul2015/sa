Задача
На основе входных данных спроектировать индексы к таблицам без антипаттернов и с учетом особенностей указанной СУБД.
————————————————————————————————
Инструкции по выполнению
Проанализируй контекст.
На основе таблиц спроектируй индексы таблицы БД.
Для индексов:
- укажи назначение индекса;
- объясни необходимость индекса;
- не создавай индексы без обоснования.
————————————————————————————————
Правила выбора типа индекса
1. PostgreSQL
| Тип данных | Тип индекса |
|---|---|
| BIGINT / INT | BTREE |
| SMALLINT | BTREE |
| UUID | BTREE |
| VARCHAR / TEXT | BTREE |
| DATE / TIMESTAMP | BTREE |
| BOOLEAN | BTREE |
| JSONB | GIN |
| TSVECTOR | GIN |
| ARRAY | GIN |
| Геоданные (PostGIS geometry/geography) | GiST |
2. MySQL
| Тип данных | Тип индекса |
|---|---|
| BIGINT / INT | BTREE |
| SMALLINT | BTREE |
| VARCHAR / TEXT | BTREE |
| DATE / DATETIME / TIMESTAMP | BTREE |
| BOOLEAN | BTREE |
| JSON | BTREE |
| SPATIAL геоданные | SPATIAL |
Учитывать:
- FULLTEXT использовать только если явно указано;
- HASH не использовать без явного требования.
3. MS SQL Server
| Тип данных | Тип индекса |
|---|---|
| BIGINT / INT | BTREE |
| UNIQUEIDENTIFIER | BTREE |
| VARCHAR / NVARCHAR | BTREE |
| DATE / DATETIME2 | BTREE |
| XML | PRIMARY XML INDEX |
| Геоданные (geometry/geography) | SPATIAL |
4. Oracle
| Тип данных | Тип индекса |
|---|---|
| NUMBER | BTREE |
| VARCHAR2 | BTREE |
| DATE / TIMESTAMP | BTREE |
| CLOB | CONTEXT |
| JSON | SEARCH INDEX |
| SDO_GEOMETRY | SPATIAL |
5. SQLite
| Тип данных | Тип индекса |
|---|---|
| INTEGER | BTREE |
| TEXT | BTREE |
| NUMERIC | BTREE |
| REAL | BTREE |
Учитывать:
- GIN/GiST/SPATIAL отсутствуют;
- FULLTEXT только через FTS5 и только если явно указано.
