# sa — AI-ассистент для системного анализа

Веб-инструмент для системных аналитиков, который на основе описания предметной области автоматически генерирует проектную документацию с помощью LLM (GPT / Claude).

Задачи выстроены в логическую цепочку: выход одного шага становится входом следующего — от бизнес-целей до SQL-скриптов и REST API дизайна.

## Возможности

### Аналитика

| Задача | Входные данные | Особенности |
|---|---|---|
| **Контекст** | Предметная область, описание системы, стейкхолдеры, ограничения | - |
| **Бизнес-цели** | Предметная область, контекст, стейкхолдеры | — |
| **Бизнес-требования** | Предметная область, бизнес-цели, текущие проблемы | Проверка качества |
| **ФТ** | Предметная область, бизнес-требования, сущности | Проверка качества |
| **НФТ** | Предметная область, ФТ, тип НФТ, параметры нагрузки | Проверка качества |
| **User Story** | Предметная область, ФТ, стейкхолдеры | Критерия приёмки |
| **Use Case** | Предметная область, процесс, архитектура, интеграции, User Story | — |
| **Sequence-диаграмма** | Use Case | Генерирует PlantUML → SVG |

### Проектирование

| Задача | Входные данные |
|---|---|
| **Логическая модель** | Предметная область, контекст проектирования |
| **Физическая модель** | Предметная область, контекст, тип БД, НФТ |
| **Индексы** | Таблицы, тип БД, ФТ, НФТ |
| **ER-диаграмма** | Схема БД → Mermaid → SVG |

### Разработка

| Задача | Входные данные |
|---|---|
| **SQL-скрипт** | Тип БД, схема БД |
| **SQL мок-данные** | Тип БД, схема БД, количество записей |
| **REST API список** | Сервис, процесс, тип API, архитектура, авторизация, версионирование, таблицы |
| **REST API дизайн** | Процесс, тип API, формат данных, архитектура, авторизация, эндпоинт |

## Технологии

- **Python** — основной язык
- **NiceGUI** — фреймворк для веб-UI
- **FastAPI / Uvicorn** — веб-сервер
- **Anthropic API / OpenAI API** — интеграция с LLM
- **python-dotenv** — управление переменными окружения

## Установка

```bash
git clone https://github.com/aigulfaizul2015/sa.git
cd sa
pip install -r requirements.txt
```

## Настройка

Создайте файл `.env` в корне проекта:

```env
ACCESS_CODE=ваш_код_доступа
STORAGE_SECRET=секретный_ключ
MAX_HISTORY_ITEMS =максимальное кол-во записей в истории
GPT_API_KEY =ключ OpenAi
GPT_MODEL =модель OpenAi
CLAUDE_API_KEY =ключ Claude
CLAUDE_MODEL =модель Claude
```

## Запуск

```bash
python ui_main.py
```

Приложение будет доступно по адресу `http://localhost:8080`.
